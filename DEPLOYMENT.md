# Deploying Accutek Solar Website on Render

## What You Need

1. A free [Render](https://render.com) account (sign up with GitHub)
2. A free [MongoDB Atlas](https://cloud.mongodb.com) cluster (for the database)

---

## Step 1: Set Up MongoDB Atlas (5 min)

1. Go to [cloud.mongodb.com](https://cloud.mongodb.com) and create a free account
2. Click **Build a Database** → choose **M0 Free** tier
3. Pick **AWS / us-east-1** (closest to Indiana)
4. Set a database username and password — save these!
5. Under **Network Access**, click **Add IP Address** → **Allow Access from Anywhere** (required for Render)
6. Click **Connect** → **Drivers** → copy the connection string  
   It looks like: `mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority`

---

## Step 2: Deploy on Render (5 min)

### Option A: One-click Blueprint (recommended)

1. Go to [dashboard.render.com](https://dashboard.render.com)
2. Click **New** → **Blueprint**
3. Connect your GitHub account and select **LordTolan/Accutek-Solar-Website**
4. Render reads `render.yaml` and sets up both services automatically
5. Fill in the environment variables it asks for:
   - **MONGO_URL** — paste your MongoDB Atlas connection string from Step 1
   - **REACT_APP_BACKEND_URL** — leave blank for now (you'll fill this after the backend deploys)
6. Click **Apply**

Once the backend deploys, copy its URL (e.g., `https://accutek-solar-api.onrender.com`) and set it as the **REACT_APP_BACKEND_URL** in the frontend service's environment settings.

### Option B: Manual setup

**Backend:**
1. New → Web Service → connect repo
2. Root directory: `backend`
3. Build command: `pip install -r requirements.txt`
4. Start command: `uvicorn server:app --host 0.0.0.0 --port $PORT`
5. Add environment variables: `MONGO_URL`, `DB_NAME=accutek_solar`, `JWT_SECRET` (any random string)

**Frontend:**
1. New → Static Site → connect repo
2. Root directory: `frontend`
3. Build command: `yarn install && yarn build`
4. Publish directory: `build`
5. Add rewrite rule: `/*` → `/index.html` (for React Router)
6. Add environment variable: `REACT_APP_BACKEND_URL=https://your-backend-url.onrender.com`

---

## Step 3: Verify

1. Visit your frontend URL (e.g., `https://accutek-solar-web.onrender.com`)
2. Check that pages load and the contact form works
3. Test the admin login at `/admin`

---

## Step 4: Connect Your Domain (when ready)

1. In Render dashboard → your frontend service → **Settings** → **Custom Domains**
2. Add `accuteksolar.com` and `www.accuteksolar.com`
3. Render gives you DNS records to add at your domain registrar
4. Once DNS propagates (5–30 min), your site is live on your real domain

⚠️ Don't do this until you've verified the new site is working on the temp URL.

---

## Auto-Deploy

Every time code is pushed to `main`, Render automatically rebuilds and deploys both services. No manual steps needed.

---

## Free Tier Notes

- Render free tier spins down after 15 min of inactivity. First request takes ~30 sec to wake up.
- Upgrade to Starter ($7/mo per service) for always-on performance.
- MongoDB Atlas M0 is free forever with 512 MB storage — plenty for leads.

## Environment Variables Reference

### Backend
| Variable | Required | Description |
|----------|----------|-------------|
| `MONGO_URL` | ✅ | MongoDB Atlas connection string |
| `DB_NAME` | ✅ | Database name (default: `accutek_solar`) |
| `JWT_SECRET` | ✅ | Random secret for auth tokens (auto-generated by Blueprint) |
| `CORS_ORIGINS` | ✅ | Allowed origins (`*` for dev, your frontend URL for prod) |
| `SENDGRID_API_KEY` | Optional | For lead email notifications |
| `LEAD_NOTIFY_FROM` | Optional | Sender email for lead notifications |
| `LEAD_NOTIFY_TO` | Optional | Recipient email for lead notifications |

### Frontend
| Variable | Required | Description |
|----------|----------|-------------|
| `REACT_APP_BACKEND_URL` | ✅ | Your backend's public URL |
